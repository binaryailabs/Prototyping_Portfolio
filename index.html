Low-Code & Advanced Prototyping

Portfolio demonstrating how learners are guided to move from

ideas → logic → interactive digital experiences

using low-code and advanced prototyping tools.

About the Faculty

Industry practitioner with over a decade of experience across diverse technology stacks, and a Toastmasters mentor and coach guiding youth exploring careers in digital design and technology.

I have served as a Visiting Faculty at SIES College, teaching computer skills, digital tools and advance office skills. Have also conducted workshops for low code no code platforms among Toastmasters community for design prototyping, website building and digital solutions.

My teaching approach emphasizes clarity of thinking—helping learners understand logic, interaction, and system behavior rather than focusing on code complexity.

Teaching Philosophy

  • Start with intent and user experience, not tools
  • Use low-code to accelerate learning, not bypass understanding
  • Translate design decisions into logic and systems
  • Create prototypes that are testable, explainable, and maintainable

Sample Assignments – Prototyping Portfolio

Assignment 1: Low-Code MVP – Workshop Registration System

Key Areas: Low-Code Platforms · Rapid Prototyping · Database Integration · Dynamic Content

Assignment Brief:
Students design a functional MVP for a creative workshop where users can register and view confirmation details dynamically.

Prototype Focus:

  • Form-based UI connected to a simple database
  • Dynamic confirmation screen
  • Auto-updating attendee list

Learning Outcomes:

  • Understand UI → data → UI flow
  • Experience real MVP thinking
  • Differentiate static mockups from working prototypes

Tools: Glide / Webflow CMS / Google Sheets

Assignment 2: Advanced Interactive Prototype – Course Enrollment Flow

Key Areas: Advanced Prototyping · Conditional Logic · Developer Handoff

Assignment Brief:
Students create a high-fidelity prototype for a course enrollment and checkout experience, including success and error states.

Prototype Focus:

  • Conditional logic for form validation
  • Micro-interactions and loading states
  • Realistic UX transitions

Learning Outcomes:

  • Designing for real-world interactions
  • Communicating intent to developers
  • Understanding UX impact on implementation

Tools: Figma (Smart Animate, Variables)

Assignment 3: Dynamic Dashboard – Personalized Student Experience

Key Areas: Conditional Logic · Dynamic Content · User Testing

Assignment Brief:
Students design a dashboard that adapts content based on user preferences, progress, and skill level.

Prototype Focus:

  • Conditional content display
  • User input handling
  • Adaptive UI states

User Testing:

  • Task-based testing with 2 users
  • Observations documented and iterated

Learning Outcomes:

  • Designing systems, not screens
  • Testing assumptions through interaction
  • Iterating based on feedback

Tools: Figma / Glide
